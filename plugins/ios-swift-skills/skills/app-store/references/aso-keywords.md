# ASO — App Store Optimization & Keywords

## Keyword Research Methodology

### 1. Brainstorm

Start with categories:
- **Core function**: GPX, export, workout, route, GPS, track
- **Activity types**: running, cycling, hiking, walking, swimming
- **Source**: Apple Health, HealthKit, fitness
- **Use case**: share route, backup workouts, Strava upload
- **Competitor terms**: what users search when looking for similar apps

### 2. Evaluate Keywords

| Criteria | Target |
|----------|--------|
| Relevance | Must describe what your app does |
| Search volume | Medium-high (niche keywords convert better than generic) |
| Competition | Low-medium (avoid competing with Nike, Strava on generic terms) |
| Word length | Shorter keywords have more combinations |

### 3. Prioritize

Place highest-value keywords in:
1. **App Name** (30 chars) — highest search weight
2. **Subtitle** (30 chars) — second highest weight
3. **Keyword field** (100 chars) — indexed but not visible

## Keyword Field Optimization (100 chars)

### Rules
- Separate with commas, **no spaces after commas**
- Don't repeat words already in app name or subtitle
- Use singular form (Apple indexes both singular and plural)
- Don't use prepositions (to, for, the, and)
- Don't use your app name or category name
- Each word only needs to appear once (Apple combines them)

### Example for SnapGPX

App Name: `SnapGPX — Workout Exporter`
Subtitle: `Export Health Workouts to GPX`

Keyword field (don't repeat "workout", "export", "GPX", "health"):
```
running,cycling,hiking,route,fitness,tracker,gps,track,share,strava,map,elevation,pace,activity,log
```

### Cross-Localization Strategy

Keywords in other language localizations are also indexed for that market. For the US market, you can use keywords in the Spanish (Mexico) localization that also index for English-speaking users.

## Writing the Description

### Structure (4000 chars)

```
[Opening hook — 1-2 sentences, benefit-focused]

[Core features — bullet list with emoji or unicode markers]
• Export any Apple Health workout to standard GPX format
• View detailed route maps with elevation profiles
• Share routes with friends or upload to Strava
• Support for running, cycling, hiking, and 15+ activity types

[Differentiator paragraph — what makes you unique]

[Social proof — download count, ratings, testimonials]

[Closing CTA]
```

### Promotional Text (170 chars)

Updated without app review. Use for:
- Seasonal messaging
- New feature announcements
- Social proof updates

```
Export your Apple Health workouts to GPX in one tap. 
Running, cycling, hiking — see your routes, share your stats. ⛰️
```

### What's New (per release)

```
What's New in 1.2:

• Merged route preview for multi-segment workouts
• Elevation chart with gradient coloring
• 11 languages now supported
• Performance improvements for large workout lists

Have feedback? We'd love to hear from you at support@snapgpx.app
```

## Seasonal Keyword Updates

| Season | Keywords to emphasize |
|--------|----------------------|
| Jan-Feb | fitness,resolution,training plan |
| Mar-May | marathon,spring,outdoor |
| Jun-Aug | hiking,trail,vacation,travel |
| Sep-Nov | fall running,cycling season |
| Dec | year review,annual summary |

## Monitoring

- Track keyword rankings weekly (App Store Connect or third-party tool)
- Monitor impression → tap-through → download conversion
- A/B test screenshots and descriptions using Product Page Optimization
- Update keywords every 4-6 weeks based on performance data
